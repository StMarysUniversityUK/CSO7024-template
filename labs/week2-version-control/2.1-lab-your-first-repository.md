# 2.1 Lab: Your First Repository

> **Week 2 · Lesson 1 · Indicative time: 30 minutes · Supports WLO1**
> Part of the CSO7024 DevOps lab series. Companion to the Week 2 module content.

## Overview

In this lab you create your first Git repository, build up a small history of commits and use `git log` and `git diff` to inspect what you have done. The lab is designed for learners who have either never used Git before or have used it only superficially. Even if you are confident with Git already, please complete it: the repository you create here is the working directory you use throughout the rest of the week.

## Learning outcomes

By the end of this lab you will be able to:

- Initialise a Git repository and configure your commit identity.
- Move a change through the working directory, the staging area and the repository with `git add` and `git commit`.
- Inspect a repository's history with `git log` and compare versions with `git diff`.

## Before you start

- You work entirely on your own machine, using Git from the command line. You do **not** need a GitHub account yet; that comes in Lesson 3.
- If you have not installed Git, follow [`SETUP.md`](../../SETUP.md) first.
- Open a terminal. Every command below is run in that terminal.

## Steps

### Step 1 — Configure your identity

Set your name and email so that commits are attributed correctly:

```bash
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
```

### Step 2 — Create and initialise a repository

Create an empty folder called `devops-week2`, move into it and initialise it as a Git repository:

```bash
mkdir devops-week2 && cd devops-week2
git init
git status
```

Read the `git status` message: you are on the default branch with no commits yet.

### Step 3 — Make your first commit

Create a file called `README.md` containing a single line, `# DevOps Week 2 Practice Repository`, then watch how it moves from untracked to staged to committed:

```bash
git status            # README.md is untracked
git add README.md
git status            # README.md is now staged
git commit -m "Initial commit"
git log               # observe your first commit
```

### Step 4 — Make a series of further commits

Add at least three more commits. Make the changes deliberate and the commit messages descriptive. Suggested edits:

- Add a short "About" paragraph to `README.md`.
- Create a file `notes.md` and write down two things you learned about Git this lesson.
- Add a third file of your choice, for example a small Python script that prints `Hello, DevOps`.

### Step 5 — Inspect your history

```bash
git log                              # the default view
git log --oneline                    # a more compact view
git log --oneline --graph --all      # a preview of the graph view, more useful once you have branches
git diff HEAD~1 HEAD                 # the changes between the previous commit and the most recent
```

### Step 6 — Capture a screenshot

Take a screenshot of your final `git log --oneline --graph` output. You will refer back to it in the discussion activity at the end of this lesson.

## Success criteria

You have completed this lab when your `git log` shows the commits you made, in the order you made them, with descriptive messages and your name as the author.

## Stretch goal (optional)

Use `git show <commit-hash>` to inspect a specific commit, and explore `git log --stat` to see file-level statistics for each commit.

## Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| `git config` prompts for nothing and seems to do nothing | That is expected; it writes silently | Run `git config --global --list` to confirm your name and email are set |
| `Author identity unknown` when committing | Identity not set | Re-run Step 1 |
| `nothing to commit, working tree clean` | You have not edited or staged a file | Edit a file, then `git add` it before committing |
